# Coding scheme

## Degree level

Suggested normalisation:

- `BS`
- `MS`
- `PhD`
- `Diploma`
- `Other/unknown`

## Topic area

Start with broad, auditable categories:

- artificial intelligence / machine learning
- optimization / operations research
- software engineering
- data science / business intelligence
- energy
- fisheries / marine
- health / medical technology
- civil / construction
- mechanical / manufacturing
- other / unknown

## External involvement

| Code | Label                 | Description                                                                                    |
|-----:|-----------------------|------------------------------------------------------------------------------------------------|
|    0 | none visible          | No company, agency, NGO, or external institution visible in abstract/PDF snippet               |
|    1 | contextual mention    | External actor appears only as background/context                                              |
|    2 | data/source           | External data, case, or material used                                                          |
|    3 | collaboration         | Thesis appears to be done with/for an external organisation                                    |
|    4 | employment/industrial | Thesis explicitly linked to employment, industrial placement, commissioned project, or similar |

Always keep:

- evidence sentence;
- actor name, if available;
- actor type;
- confidence;
- whether the label was rule-based, LLM-assisted, or manually validated.
